Toast

info/success/warning/error

Overview

토스트는 사용자 액션에 대한 짧은 피드백을 화면 우하단에 잠깐 띄웠다가 자동으로 사라지는 알림이다. 저장 완료, 삭제, 에러처럼 일회성 알림에 쓰고, 사용자 확인이 꼭 필요한 내용은 Modal을 쓴다. js/main.js의 showToast()가 실제 DOM을 만들고 4초 뒤 자동으로 제거한다.

Variants

클래스 설명
toast 기본형 (info). 회색 아이콘
toast--success 성공. accent 색상 아이콘
toast--warning 경고. warning 색상 아이콘
toast--error 에러. danger 색상 아이콘
toast [toast--{success|warning|error}] — showToast()가 생성하는 실제 마크업
알림
새로운 업데이트가 있습니다.
저장 완료
변경 사항이 저장되었습니다.
저장 용량 임박
곧 저장 공간의 90%에 도달합니다.
저장 실패
네트워크 오류가 발생했습니다. 다시 시도해주세요.

Options

속성 설명
desc 선택. showToast({ title, desc })에서 생략하면 toast__desc가 만들어지지 않는다 (title만 표시)
desc 유무
제목만
제목 + 설명
desc를 넘기면 이 줄이 추가된다.

States

생성 직후에는 opacity:0 상태였다가 다음 프레임에 is-visible가 붙으면서 페이드인된다. toast__close 클릭 또는 4초 경과 시 is-visible가 빠지면서 페이드아웃되고, 트랜지션이 끝나면 DOM에서 완전히 제거된다. 직접 눌러서 확인하려면 아래 Examples의 버튼을 클릭한다.

Examples

data-toast="{success|warning|error}" data-toast-title="..." data-toast-desc="..." — 버튼을 눌러보면 실제로 우하단에 뜬다

Guidelines

✓ 권장

  • 저장/삭제 등 일회성 피드백에만 사용
  • 제목만 보고도 결과를 알 수 있게 간결하게 작성
  • 에러의 원인이 불명확하면 desc에 다음 행동 안내 (예: "다시 시도해주세요")

✕ 지양

  • 사용자 확인이 꼭 필요한 내용 (Modal 사용)
  • 토스트 안에 폼 입력 요소나 복잡한 인터랙션 넣기
  • 4초 안에 다 읽기 힘든 긴 설명문
