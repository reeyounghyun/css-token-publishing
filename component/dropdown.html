Dropdown

셀렉트, 커스텀 드롭다운 5종, disabled 항목, 구분선

Overview

Dropdown은 여러 옵션 중 하나를 골라 트리거에 표시하는 컨트롤이다. 단순한 목록엔 네이티브 select를 쓰고, 아이콘·아바타·상태점처럼 옵션을 꾸며야 하면 커스텀 드롭다운(dropdown-custom)을 쓴다. 프로필/메뉴 목적이면 Dropdown Menu 컴포넌트를 대신 참고한다.

Variants

구현 설명
select 네이티브 select. 스타일링 제약은 있지만 접근성·모바일 대응이 가장 안정적
dropdown-custom 커스텀 구현. 아이콘/아바타/상태점 등 자유롭게 꾸밀 수 있지만 data-dropdown-custom JS가 필요
select (네이티브) vs dropdown-custom[data-dropdown-custom] > dropdown__trigger, dropdown__list > dropdown__item

Options

클래스/속성 설명
아이콘 선택. 트리거/옵션에 svg를 추가해 항목을 구분
dropdown__avatar 선택. 옵션 앞에 원형 프로필 이미지
dropdown__divider 선택. 옵션 사이에 구분선을 넣어 그룹 분리
aria-disabled="true" 선택. 클릭해도 선택되지 않는 비활성 옵션 (dropdown__item에 부여)
dropdown__divider + dropdown__item[aria-disabled="true"]

States

속성 설명
aria-expanded="true" 열림 상태. 트리거 클릭 시 data-dropdown-custom JS가 토글
aria-selected="true" 선택된 옵션. dropdown__check 체크마크가 표시됨
is--active 키보드로 이동 중인 항목 (hover와 동일한 모양)
dropdown__item[aria-selected="true"] > dropdown__check

Guidelines

✓ 권장

  • 선택된 항목은 체크마크나 굵은 글씨로 명확히 표시
  • 커스텀 드롭다운도 role/aria-* 속성을 네이티브 select만큼 갖추기
  • 옵션이 많으면(20개 이상) 검색 가능한 형태 고려 (이 컴포넌트는 검색 없음)

✕ 지양

  • 옵션이 2~3개뿐인데 드롭다운 사용 (Radio가 더 빠름)
  • 커스텀 드롭다운에서 aria-expanded/role 속성 생략
  • 더 이상 쓰이지 않는 .dropdown 클래스 재사용 (.dropdown-custom만 사용)
