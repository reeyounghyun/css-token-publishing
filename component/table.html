Overview

Table은 여러 행/열로 이뤄진 데이터를 보여주는 컴포넌트다. 시맨틱 <table> 마크업이 기본이고, sticky 헤더/첫 컬럼 고정처럼 복잡한 레이아웃이 필요하면 <div> + CSS Grid로 만든 Grid Table을 대신 쓸 수 있다 (동일한 시각 결과, 접근성 role은 직접 부여해야 함).

Variants

구현 설명
table 시맨틱 <table> 마크업. 기본값으로 권장
grid-table <div> + CSS Grid. role="table/row/cell/columnheader"를 직접 부여해야 접근성이 동등해짐
table vs grid-table > grid-table__row(--header), grid-table__cell — 같은 데이터, 다른 마크업

Guidelines

✓ 권장

  • 가능하면 시맨틱 <table> 사용, Grid Table은 sticky 등 table로 어려운 경우에만
  • 빈 결과는 안내 문구로, 로딩 중엔 스피너로 명확히 구분
  • 정렬 가능한 헤더는 aria-sort를 실제 정렬 상태와 동기화

✕ 지양

  • 행이 수백 개인데 페이지네이션/가상스크롤 없이 전부 렌더링
  • Grid Table 쓰면서 role 속성 생략 (스크린리더가 표로 인식 못함)
  • 피벗 테이블에서 첫 컬럼 고정 없이 가로 스크롤만 시키기 (행 제목을 잃어버림)
