Empty State

기본/검색결과 없음/에러

Overview

빈 상태(Empty State)는 표시할 데이터가 없을 때 사용자에게 상황을 설명하고 다음 행동을 안내하는 컴포넌트다. 목록이 비어있거나, 검색 결과가 없거나, 데이터를 불러오는 데 실패했을 때 빈 화면 대신 사용한다.

Variants

클래스 설명
empty-state 기본형. 정보 안내용, 회색 아이콘
empty-state--error 에러/실패 상태. 아이콘만 위험 색상으로 강조
empty-state [empty-state--error]

제목

상황을 설명하는 문구가 들어갑니다.

제목

상황을 설명하는 문구가 들어갑니다.

Options

클래스 설명
empty-state__action 선택. 사용자가 취할 수 있는 다음 행동이 있을 때만 버튼 추가, 없으면 생략
empty-state__action 유무

액션 없음

그냥 상황만 안내할 때는 버튼을 생략한다.

액션 있음

할 수 있는 행동이 있으면 버튼을 추가한다.

States

정적 안내 컴포넌트라 hover/disabled 같은 상태 클래스가 따로 없다. 상황(기본/에러)과 옵션(액션 유무) 조합만으로 화면이 결정된다 — 실제 조합 예시는 아래 Examples 참고.

Examples

빈 목록 안내 (액션 버튼 있음)

아직 항목이 없습니다

새 프로젝트를 추가해서 팀 작업을 시작해보세요.

검색 결과 없음 (액션 버튼 생략)

검색 결과가 없습니다

"마케팅 대시보드"와 일치하는 항목을 찾지 못했습니다. 다른 검색어로 시도해보세요.

에러 / 불러오기 실패 (empty-state--error)

데이터를 불러오지 못했습니다

네트워크 오류가 발생했습니다. 잠시 후 다시 시도해주세요.

Guidelines

✓ 권장

  • 상황을 명확히 설명하는 제목 + 다음 행동을 안내하는 설명문
  • 할 수 있는 행동이 있을 때만 empty-state__action 추가
  • 에러 상태는 empty-state--error로 시각적으로 구분

✕ 지양

  • 모든 상황에 "데이터가 없습니다" 같은 똑같은 문구 재사용
  • 액션 버튼 라벨을 "확인"처럼 모호하게 쓰기 — 구체적인 동사로
  • 에러 문구에 스택트레이스 등 기술적인 원문 그대로 노출
