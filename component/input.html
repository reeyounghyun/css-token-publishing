Input

비밀번호/이메일/전화번호/기본 텍스트, 사이즈 3종

Overview

Input(텍스트 필드)은 사용자로부터 짧은 텍스트를 입력받는 컴포넌트다. 이메일·전화번호처럼 형식이 정해진 입력엔 전용 레이아웃(아이콘, 국가코드)을 붙이고, 여러 줄이 필요하면 Textarea를 쓴다.

Variants

클래스 설명
field 기본 텍스트 입력
field--password 비밀번호. 눈 아이콘으로 마스킹 토글
field__number 전화번호. 왼쪽에 국가 코드 선택 버튼(lang__box) 추가 — 기존 네이밍이 field--가 아니라 field__라 예외적이지만 실제 쓰이는 클래스라 그대로 둠
field [field--password | field__number]

Options

클래스 설명
field--sm · field--lg 크기. 기본은 md(미지정)
icon 선택. field__box 맨 앞에 넣으면 입력 왼쪽에 아이콘 표시 (이메일 등)
field--{sm|lg} / field__box > icon(선택)
아이콘

States

클래스/속성 설명
field--disabled 비활성. 회색 배경, 입력 커서 not-allowed
readonly 수정 불가하지만 값은 보임 (네이티브 속성, disabled와 달리 값이 폼 제출에 포함됨)
field__error 검증 실패. 빨간 테두리 + 힌트 텍스트 색상 변경 — field--가 아닌 예외 네이밍
field [field--disabled] [readonly] [field__error]
아이콘

올바른 이메일 형식이 아닙니다.

Examples

로그인 폼 — 이메일 + 비밀번호 조합
아이콘

Guidelines

✓ 권장

  • 필수 안내는 label/힌트 텍스트에, placeholder는 예시로만
  • 에러 시 힌트 텍스트를 구체적인 안내로 교체
  • 비밀번호는 기본적으로 마스킹, 눈 아이콘으로만 토글 제공

✕ 지양

  • label 없이 placeholder만으로 용도 설명 (스크린리더 접근성 저해)
  • 여러 줄 입력이 필요한데 Input 사용 (Textarea를 써야 함)
  • 새 필드에 field__error/field__number 같은 예외 네이밍을 새로 복제 (신규 옵션은 field-- 형태로)
