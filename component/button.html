버튼, SNS 로그인 버튼, 로딩 상태

Overview

버튼은 사용자가 화면에서 취할 수 있는 동작을 트리거하는 요소다. btn--primary는 화면의 가장 중요한 동작 하나에만 쓰고, 나머지 동작은 btn--default/btn--dashed/btn--text/btn--link 중 위계에 맞는 걸 고른다. 실제 조합 예시(아이콘, SNS 로그인, 버튼 그룹)는 아래 Examples를 참고한다.

Variants

클래스 설명
btn--primary 채워진 버튼. 주요 액션, 화면당 1개 권장
btn--default 외곽선 버튼. 기본 액션
btn--dashed 점선 버튼. 추가/생성 유도
btn--text 배경 없는 버튼. 보조 액션
btn--link 링크형 버튼. 내비게이션 성격

Options

클래스 설명
btn--smbtn--lg 크기. 기본은 md(미지정)
btn--round 필(pill) 형태
is-danger 위험한 동작 강조. btn--text와 조합해서만 사용 (색만 덮어씀)

Examples

Icon

btn btn--{primary|default} > svg (자식으로 넣으면 gap 토큰으로 자동 정렬)

Button Group

btn btn--{default|primary} 나란히 배치 (취소/확인형 액션 그룹)

SNS 로그인

sbtn sbtn--filled sbtn--{google|facebook|apple} [sbtn--icon]
sbtn sbtn--outline sbtn--{google|facebook|apple} [sbtn--icon]
sbtn sbtn--mono sbtn--{google|facebook|apple} [sbtn--icon]

Guidelines

✓ 권장

  • 버튼 라벨은 동사로 (예: "저장하기")
  • Primary 버튼은 화면당 1개만 사용
  • 아이콘은 svg를 자식으로 넣으면 자동 정렬
  • 로딩 시 라벨을 span.btn__label로 감싸기

✕ 지양

  • 한 화면에 primary 버튼 여러 개
  • 크기 토큰 대신 임의 padding 지정
  • disabled 버튼 클릭 이벤트에 로직 의존
