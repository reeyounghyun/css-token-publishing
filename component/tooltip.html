Tooltip

4방향, JS 없이 동작

Overview

툴팁은 아이콘·텍스트에 마우스를 올리거나 키보드로 포커스했을 때 짧은 보조 설명을 보여주는 요소다. 모바일에는 hover가 없으므로 필수 정보는 담지 않고, 있으면 도움이 되는 부가 설명에만 쓴다. :hover/:focus-within만으로 동작해서 JS가 필요 없다.

Variants

클래스 설명
tooltip--top 위쪽에 표시 (기본값, 클래스 생략해도 동일)
tooltip--bottom 아래쪽에 표시
tooltip--left 왼쪽에 표시
tooltip--right 오른쪽에 표시
tooltip tooltip--{top|bottom|left|right} — 마우스를 올리거나 Tab으로 포커스
위쪽에 뜨는 툴팁입니다. 아래쪽에 뜨는 툴팁입니다. 왼쪽에 뜨는 툴팁입니다. 오른쪽에 뜨는 툴팁입니다.

Options

방향 외 독립적인 수정자 클래스는 없다. 트리거 안에 아이콘을 넣을지 텍스트를 넣을지는 클래스가 아니라 자식 콘텐츠 구성으로 결정한다 (아래 Examples 참고).

States

열림/닫힘을 나타내는 별도 클래스가 없다. :hover/:focus-within일 때만 tooltip__content가 보인다 (기본은 opacity:0; visibility:hidden). 위 Variants 데모에 직접 마우스를 올리거나 Tab으로 이동해서 확인할 수 있다.

Examples

tooltip__trigger에 아이콘 대신 텍스트 — 밑줄 점선으로 힌트 표시
프로 플랜에서만 사용할 수 있습니다.
비활성 버튼에 이유 설명 — disabled 버튼은 자체 hover가 안 먹으므로 감싸는 span에 툴팁을 건다
내보내기 권한이 없어 내보낼 수 없습니다.

Guidelines

✓ 권장

  • 한 줄 이내의 짧은 보조 설명만
  • 트리거에 aria-describedby로 tooltip__content의 id 연결
  • 아이콘 트리거는 스크린리더용 텍스트(aria-label 등) 함께 제공

✕ 지양

  • 툴팁 안에만 있는 필수 정보 (모바일은 hover가 없어 아예 못 봄)
  • 툴팁 안에 링크/버튼 등 인터랙티브 요소 넣기 (hover 벗어나면 닫혀서 클릭 불가)
  • 긴 문단 텍스트 (max-width 220px 안에서 줄바꿈이 어색해짐)
